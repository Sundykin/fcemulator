## Why

当前仓库已建成一台高完成度的自研 NES 模拟器(CPU/PPU/APU/11 个 Mapper/存档/金手指,~90%)、现代播放器+调试器 UI(fc-tauri,6 张设计稿全实现)、以及硬件层 MCP(fc-mcp + tauri-bridge)。**护城河已经挖好**,但《策划案-基于现状-可行性评估.md》愿景里约 70% 篇幅的"创作工具链"仍是 0:没有工程管理、代码编辑器、汇编/编译、CHR/地图/音频编辑、工程层 MCP。

文档已拍板**集成优先(v2 终版)**战略:不再追求"零依赖全自研",而是把 cc65 / FamiStudio / CodeMirror 等社区事实标准**无缝粘合**成 AI 原生的统一工作流,自研只留在护城河(模拟器/调试器/构建编排器/工程层 MCP/AI 编排)。这把整体可行性从 12–24 人月降到 ~3–4 人月。

本变更把这条战略**固化为可执行的里程碑计划**,并把第一阶段 **M1 · 创作最小闭环**细化为可立即开工的规格与任务——一旦交付,已有的模拟器/调试器/MCP 立即变现为"能写、能编、能跑、能调"的一栈式 IDE 雏形。

## What Changes

**本变更是"里程碑计划 + M1 可实施规格"两部分:**

- **战略与里程碑路线固化**(proposal/design):确立"集成优先 + 护城河变现"为产品主线,定义 M1→M4 四阶段及验收口径、关键技术决策(汇编器封装 ca65 sidecar、音频先内置精简 tracker + FamiStudio 文件对接、编辑器用 CodeMirror 6、沿用 Vue3+Pinia、务实混合"零依赖")。
- **M1 创作最小闭环细化为规格 + 任务**(specs/tasks),目标:**新建工程 → 写汇编 → 一键编译/链接 → 打包 .nes → 内嵌运行 → 调试**,且编译报错 ↔ 代码行、断点 ↔ 代码行双向跳转。
  - 新增 Rust 编排层(fc-tauri `src-tauri/`)+ Vue3 IDE 外壳(文件树/编辑器/构建面板)。
  - 引入 cc65(ca65/ld65)作为 Tauri **sidecar**,捆绑链接脚本 `.cfg` 与头/库环境变量。
  - 复用已有 `ControlDeck` 内嵌运行 + 调试器(不重写)。
- **M2–M4 作为里程碑级占位**写入 tasks(不在本变更内细化为规格):
  - **M2 · 资源 + 音频(~4–6 人周)**:内置 CHR/地图轻量编辑器 + FamiStudio CLI 导出/引擎模板对接 + 格式转换器。
  - **M3 · AI 一栈式(~4–6 人周)**:工程层 MCP + AI 编排(新建工程/编码/编译排错/硬件调试)+ "内核即校验"闭环 + AI 面板。
  - **M4 · 打磨/生态**:跨平台打包签名、模板/案例库、"外部 pro 工具打开+回写"、更多 Mapper/FDS、自定义智能体接入。

里程碑总览:

| 里程碑 | 目标 | 工作量 | 本变更细化程度 |
|---|---|---|---|
| **M1 创作最小闭环** | 工程→编辑→cc65 编译→打包→内嵌运行→调试,报错/断点↔代码行 | ~4–5 人周 | ✅ 规格 + 任务 |
| M2 资源 + 音频 | 内置 CHR/地图编辑器 + FamiStudio 文件对接 + 转换器 | ~4–6 人周 | ⏳ 里程碑占位 |
| M3 AI 一栈式 | 工程层 MCP + AI 编排 + 内核即校验 + AI 面板 | ~4–6 人周 | ⏳ 里程碑占位 |
| M4 打磨/生态 | 打包签名 + 模板/案例库 + 更多 Mapper/FDS + 自定义智能体 | 持续 | ⏳ 里程碑占位 |

非目标(Non-goals):本变更不自研汇编器/编译器(封装 cc65)、不自研对标 Famitracker 的音频编辑器、不重写编辑器内核、不打包 FamiStudio;"完全自研零依赖"不作为 V1 硬约束。

## Impact

- **新增/重点**:`fc-tauri/src-tauri/` 新增 Rust 编排层(工程模型、构建编排器、sidecar 进程管理、日志解析);`fc-tauri/src/` 新增 IDE 外壳(文件树/编辑器/构建面板,Pinia store + dockview/分栏)。
- **复用**:`fc-core` `ControlDeck`(内嵌运行 + 调试器 + 存档)保持 IO-free,不改核心;`fc-tauri` 现有播放器/调试面板作为内嵌运行子集。
- **外部依赖(集成)**:cc65 工具链(ca65/ld65,zlib 许可,可商用闭源打包)作为捆绑 sidecar,按 target-triple 预编译;捆绑 `.cfg` 链接脚本。
- **构建/发行**:引入跨平台 sidecar 二进制打包流程(M4 再做 macOS 签名/公证);新增"第三方许可"清单(cc65 zlib 等)。
- **后续解锁**:工程模型与构建编排器是 M2(资源对接)、M3(工程层 MCP / AI 编排 / 内核即校验)的技术内核。
