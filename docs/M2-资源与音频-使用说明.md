# M2 · 资源 + 音频 —— 使用说明

> OpenSpec 变更 `nes-ide-resources-audio`。在 M1 创作闭环之上补齐**图形(CHR)、
> 地图、音乐**资源能力。战略:护城河自研(用自研 APU 内核做 tracker 试听/校验),
> 卡脖子借力(FamiStudio 文件对接,不打包)。

## CHR 图形编辑器
- 文件树点开 `.chr`(或工具栏新建),在「CHR」面板编辑 8×8 图块:铅笔/擦除/填充/取样、
  旋转、翻转、按像素平移,撤销/重做,4 色调色板(双击色块从 NES 64 色选色),图块表总览点击切换。
- 主编辑区会随面板空间自动放大,点击/拖拽仍映射回 8×8 逻辑像素;图块表也会按可用宽度放大,
  不需要在袖珍原始像素尺寸里操作。
- 右键、Shift 或 Alt 可临时擦除到槽 0;常用快捷键:`B/P` 铅笔,`E/X` 擦除,`F` 填充,
  `I` 取样,`0-3` 选色槽,`[`/`]` 切换图块,`Q/W` 逆/顺时针旋转,
  `Shift+H/V` 水平/垂直翻转,`J/K/L/,` 将当前图块左/上/右/下平移一像素,
  对平移快捷键加 `Shift` 会循环卷绕边缘像素;Ctrl/⌘+C/V 复制/粘贴当前图块,
  Ctrl/⌘+D 复制到下一个图块,Ctrl/⌘+Z/Y 撤销/重做,Ctrl/⌘+S 保存。
- 保存导出 `chr/*.chr`(NES 2bpp planar)并登记进 `project.toml`;asm 用
  `.incbin "chr/x.chr"` 放入 `CHARS` 段即进 ROM。
- `横版`/`演示` 模板默认已有 `chr/sprites.chr`,源码会 `.incbin` 它;直接在 CHR 编辑器改完保存、重新运行即可更新 ROM 图形。

## 地图/命名表编辑器
- 文件树点开 `map/*.bin`,在「地图」面板为地图绑定 `chr/*.chr` 图块来源;绑定会写入
  `project.toml` 的 `map_chr` 表,重开工程后自动恢复。
- 可编辑 **图块层**、**属性层**(每 2×2 块 0–3 调色板)、**碰撞层**;支持缩放/网格、
  1×1/2×2/4×4 刷子、矩形填充、连续区域填充、地图取样、右键或 Shift 擦除、
  撤销/重做。
- 小地图会自动放大到可操作区域,工具栏缩放值作为最小缩放;点击位置会转换回逻辑地图格,
  保存的 `map/*.bin` 数据仍是原始 tile/attribute/collision 数组。
- 图块层支持「选区」工具复制矩形区域,Ctrl/⌘+C 复制、Ctrl/⌘+V 在当前格粘贴。
- 选中区域后,上下文栏会显示可见的「填充选区」「清空」动作;图块层还会显示「向右重复」「向下重复」,
  适合快速铺平台、墙体或重复装饰。
- 大地图可用空格拖拽或中键拖拽平移,Ctrl/⌘ + 滚轮以鼠标位置为中心缩放。
- 常用快捷键:`1/2/3` 图块/属性/碰撞层,`B/R/F/I` 刷子/矩形/填充/取样,
  `S` 选区,`G` 网格,`[`/`]` 缩放,方向键移动当前地图焦点/单格选区,
  `Shift+方向键` 扩展选区,`Alt+方向键` 平移整个选区框,
  `Ctrl/⌘+方向键` 移动选区内的图块内容并清空原位置,
  `Ctrl/⌘+Shift+方向键` 将选区内容复制到相邻同尺寸区域,
  `Enter` 用当前图块/属性/碰撞值填充选区,`Shift+Enter` 清空选区,Ctrl/⌘+S 保存地图。
- 导出 `.bin`(文档化布局:`w,h` + 命名表 + 属性 + 碰撞),asm `.incbin` 进 ROM。
- `横版`/`演示` 模板默认已有 `map/room.bin`,源码会读取其 tile 层写入 nametable;地图编辑器保存后重新运行即可更新背景布局。
- 同一模板还会读取 `map/room.bin` 的碰撞层;在地图编辑器切到「碰撞」把格子设为「挡」后保存并重新运行,角色移动会被这些格子阻挡。

## 资源转换器
- 工具栏「导入PNG」:≤4 色、边长 8 倍数的 PNG → `chr/*.chr`(超色/尺寸非法会明确报错)。
- 工具栏「导入Tiled」:Tiled CSV/JSON 导出 → `map/*.bin`。
- 智能体也可以通过 `fc-ide` MCP 的 `ide_read_chr`/`ide_write_chr`、
  `ide_read_map`/`ide_write_map` 与 `ide_read_song`/`ide_write_song`
  直接读写这些资源;再调用 `ide_open_resource` 即可让 Tauri IDE 打开并聚焦对应编辑器。
  新资源骨架可用 `ide_create_resource` 创建:`kind=source|chr|map|music`,可选 CHR 图块数、
  地图宽高/CHR 绑定、乐曲 Pattern 行数。
  需要小粒度迭代资源时,用 `ide_patch_source` 按行修改源码,用 `ide_patch_chr_tile` 修改单个 CHR 图块,
  用 `ide_patch_chr_pixels` 修改一个或多个 CHR 像素,用 `ide_transform_chr_tile`
  对单个 CHR 图块做旋转、翻转或像素平移,或用
  `ide_patch_map_cells` 修改地图图块/属性/碰撞格,用 `ide_patch_song_cell` 修改单个
  tracker Pattern 单元格,或用 `ide_patch_song_cells` 批量写入多格/短旋律。需要检查具体位置时,用
  `ide_focus_resource` 定位到源码 `line`、CHR `tile`、地图 `x/y/layer` 或音乐 `pattern/row/channel`。
  `ide_get_state.ui.active_editor` 会返回真实 IDE 当前编辑器的语义焦点,`ui.resource_history` 会返回当前是否可后退/前进、上一/下一资源以及去重后的 `recent` 最近资源,智能体无需用 DOM 桥推断当前图块、
  地图格或 tracker 单元格。资源打开/聚焦后可用 `ide_wait_ui_context` 等待前端快照确认,避免读到上一帧的
  异步状态。焦点确认后可用 `ide_patch_active_context` 直接修改当前可见上下文:源码默认当前行,
  若源码编辑器有可见选择范围,传 `scope=selection` 会按 `ui.active_editor.selection.line0/line1`
  替换整段行范围,并可用 `selection_line0/selection_line1` 等待该选择范围;
  CHR 默认当前图块;若传 `x/y/value` 则会修改当前图块的指定像素,缺省 `x/y` 可来自真实编辑器的
  `hover_pixel`,地图默认 `focus_cell` 当前格/图层,tracker 默认当前 Pattern 单元格。
  地图可显式传 `scope=brush` 使用当前画笔尺寸批量写格,或 `scope=selection` 写当前选区。
  音乐可显式传 `scope=phrase` 和 `notes` 从当前 Pattern 行/声道开始写短旋律,或传
  `scope=selection` 将 `note/instrument/volume/fx/param` 批量写入当前可见 Pattern 选区。
  `ide_patch_map_cells` 和地图 active-context 批量写入会把修改矩形回传给前端并保留为可见选区;
  需要等待区域稳定时,`ide_wait_ui_context` 可匹配 `selection_x0/y0/x1/y1`。
  tracker 批量/短句写入也会把 Pattern 范围回传给前端并高亮为可见选区;
  可用 `selection_row0/row1/channel0/channel1` 等待该短句范围。
  `ide_transform_chr_tile` 的 `op` 支持 `rotate_cw`/`rotate_ccw`/`flip_h`/`flip_v`/
  `shift_left`/`shift_right`/`shift_up`/`shift_down`;平移时传 `wrap=true` 会循环卷绕边缘像素。
  前端会收到更新事件并刷新当前工程状态。

## 内置 2A03 tracker(自研,驱动内核 APU)
- 工具栏「新建乐曲」或打开 `.song.json`,在「音乐」面板:
  - **音序/Pattern**:`Z–M` 键入音符、`1` 音符停止、`⌫` 清除;方向键移动。
  - **卷帘**:工具栏「卷帘」切换,点击放置/清除选中通道的音符,±八度。
    卷帘画布会按父级工作区重新计算行宽/音高,再把点击位置转换成 pattern 行与音符。
    支持拖拽连续写入,右键、Shift 或 Alt 拖拽擦除。
  - **编辑历史**:音符输入、卷帘绘制、效果和乐器参数支持撤销/重做;Ctrl/⌘+S 保存,
    Ctrl/⌘+Z/Y 撤销/重做,`R` 切换音序/卷帘,`[`/`]` 调整输入八度。
  - **乐器**:占空比 + 音量/琶音逐帧包络。
  - **效果**:选中单元格可设琶音 `0xy`(逐帧 base/+x/+y)。音量列 = 音量轨。
  - **试听**:用**自研 `fc-core` APU 内核**逐帧渲染(Web Audio 播放)——这是别家没有的
    "内核即试听/校验"。
- **导出**:工具栏「导出」→ `music/<name>.s`(逐帧 APU 寄存器流,与试听同源)+ 捆绑
  极简 6502 引擎 `music/fc_player.s`。主程序 `jsr fc_player_init`(reset)、
  `jsr fc_player_tick`(NMI)即在 ROM 内播放(已在内核 live 验证发声)。
- 智能体通过 `ide_write_file` 写入 `music/*.s` / `.asm` 时会自动登记为音乐构建输入,
  下一次 `ide_build` 会汇编并链接它。
- 智能体需要一个可玩的起点时可调用 `ide_scaffold_game`:它会创建 demo/horizontal 游戏工程、
  生成 `.song.json`,导出 `music/*.s` + `music/fc_player.s`,接入源码,并可选构建/运行。
- 智能体通过 `ide_patch_song_cell` 修改 `.song.json` 时,只需传 `path/pattern/row/channel`
  和要改变的 `note/instrument/volume/fx/param` 字段;Tauri IDE 会重载乐曲并选中该单元格。
  需要写旋律时用 `ide_patch_song_cells`:可传 `cells:[{row,channel,note,...}]` 精确批量修改,
  或传 `notes:["C4","D4","E4","G4"]`、`start_row`、`start_channel`、`row_step`
  一次写入短句。音符支持数字、`C4`/`C#4`/`Db4`、`...` 空音、`===` note-off;
  顶层 `instrument/volume/fx/param` 会作为批量默认值。写入后真实 Tracker 会聚焦第一处修改,
  并高亮这次写入的 Pattern 行/声道范围。已有可见选区后,可用
  `ide_patch_active_context { "kind":"music", "scope":"selection", "volume":12 }`
  直接把字段批量写入该范围。
- 智能体完成 `.song.json` 编曲后应调用 `ide_export_song`,由 Tauri 内嵌 IDE MCP 生成
  `music/<name>.s` 和 `music/fc_player.s`,写回 `project.toml` 并刷新可见 IDE。
- 导出后可继续调用 `ide_wire_song_player`(默认 `src/main.s`),由 Tauri 内嵌 IDE MCP
  幂等补齐播放器 `.import`、`reset` 初始化调用和 `nmi` 每帧 tick 调用,并让真实 IDE 聚焦源码插入行。
- 构建/运行后可调用 `ide_verify_game`,让 IDE MCP 直接从可见 Tauri 预览读取运行态、画面统计和输入响应,
  用于证明资源变更确实进入了可玩的 ROM。验证结果会同步到 IDE 顶栏验证芯片与
  `ide_get_state.ui.game_verify`;若之后又构建或运行,该状态会标为 `stale`。
- 手动工作流中,点击顶栏手柄验证芯片或 Build「体检」里的 `游戏验证` 动作会复用同一验证路径,
  适合改完 CHR/地图/音乐后确认资源已经进入当前可玩 ROM。
- **导入**:工具栏「导入FTM」可导入 FamiTracker 文本导出(2A03 五通道,基础保真)。

## FamiStudio 对接(可选,专业音频)
见 [FamiStudio-导出规范.md](FamiStudio-导出规范.md):导出 CA65 → 「导入音乐」→ `music/`
登记 → 纳入构建。

## 改资源即重建(文件监听)
工具栏「监听」开启后,`src/ chr/ map/ music/` 变更去抖后**自动重建并刷新内嵌运行**
(与手动构建串行化)。在外部工具改完资源重新导出覆盖,IDE 即自动重链、可试听/查看。

## 资源快速切换与回退
顶栏「资源」或 `Cmd/Ctrl+P` 会打开工程资源快速切换器。列表来自 `project.toml` 中登记的源码、
CHR、地图和音乐资源,并显示地图绑定的 CHR、CHR 被几个地图使用等上下文。选择资源后会复用正常
文件树打开路径,因此 Map/CHR/Tracker/源码编辑器的焦点、滚动和活动资源状态保持一致。音乐资源中
`.song.json` 会进入 Tracker;`music/*.s` / `.asm` 是音乐构建输入,会进入源码编辑器但仍标记为音乐资源。
快速打开、文件树、Map↔CHR 跳转和 MCP 资源聚焦都会进入同一套资源历史。顶栏小箭头或
`Cmd/Ctrl+[`、`Cmd/Ctrl+]` 可在最近资源之间后退/前进,适合源码→地图→CHR→音乐的来回创作。
重复打开同一个资源会更新它的最新位置,不会在历史栈里堆出多个旧副本。快速切换器空查询时会先列出最近资源,
并显示可恢复的位置摘要;输入查询时仍搜索完整的 `project.toml` 资源清单。

## 已知限制(M2)
- tracker 为**精简版**:不做扩展音频芯片(VRC6/VRC7/FDS/MMC5/N163/5B)、不做
  NSF/二进制 FTM 导入;效果目前仅琶音(滑音/颤音后续);卷帘为"够用"版;DPCM 预览静音。
- 导出引擎 `fc_player.s` 用 ZEROPAGE 段 3 字节(用 ca65 ZP 标签的工程不会冲突)。
- CHR 当前随 asm `.incbin` 进 ROM;独立 CHR-bank 管理后续。
- 文件监听为"受影响即全量重链"的保守增量。
