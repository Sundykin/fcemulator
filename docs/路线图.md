# fc 模拟器路线图 —— 向成熟模拟器 + AI 原生平台演进

> 一套 `fc-core`(纯 Rust 核心)同时输出三种形态:**CLI**(无头自动化/测试)、
> **桌面 GUI**(egui 低延迟 / Tauri+Vue 现代化界面)、**MCP 服务端**(AI 原生)。
> 本文件规划从"能玩"到"成熟模拟器 + AI 逆向/陪玩平台"的演进。

## 0. 现状(已完成)

- **内核**:周期驱动 CPU(nestest 通过)、真流水线 PPU(背景/精灵/sprite-0/滚动分屏,
  已修精灵 Y off-by-one)、APU 五声道含 **DMC DPCM**、Mapper 0/1/2/3/4(MMC3 A12 IRQ,
  blargg mmc3_test 3/5)/7、全量存档、电池 SRAM。
- **前端**:CLI(run/test/disasm/info/dbg/mcp,--shot/--wav)、egui GUI、
  **Tauri+Vue+PixiJS** 现代界面(设备像素整数缩放、cpal/WebAudio、音频时钟节拍)。
- **输入**:键盘(序号去乱序 + 每帧心跳防丢 + 失焦清键)+ **gilrs 手柄后端原生轮询**(无 IPC)。
- **MCP**:10 个工具(按键/读写内存/状态/步进/截图 PNG/存读档/复位/反汇编)。

## 2. 金手指 Cheats(`fc-core` + Tauri UI)

- **核心**:`fc-core` 新增 `cheat` 模块 —— `Cheat { addr:u16, value:u8, compare:Option<u8>, enabled:bool }`。
  ControlDeck 每帧 `run_frame` 后遍历启用的金手指写入 RAM(compare 命中才写)。
- **Game Genie 解码**:6/8 位 GG 码 → (addr, value, compare),内置标准解码表。
- **MCP/GUI 共用**:已有 `write_memory`,扩展为持久金手指列表。
- **Tauri UI**:`n-data-table`(GG码/地址/值/描述/开关),导入导出 `.cht`。

## 3. 断点调试 Debugger(`fc-core` 强化 + 面板)

- **核心**:`Debugger { breakpoints, watchpoints, trace_ring }`,在 CPU/总线钩子里判定:
  - 断点类型:执行 PC=addr、读 addr、写 addr、扫描线 N、命中条件表达式。
  - 命中即暂停 worker(`paused=true`),暴露单步/步过/运行到下一帧/运行到返回。
  - 执行追踪环(PC/操作码/寄存器/周期/扫描线),可导出。
- **反汇编**:已有 256 项表;增加跟随 PC、符号/标签、交叉引用。
- **Tauri UI**:反汇编窗(断点点击设置)、寄存器、十六进制内存编辑器(可写)、
  调用栈、PPU 查看器(复用 `dbg_pattern/dbg_nametable/dbg_oam`)。

## 4. 存档列表 + 截图(存档管理)

- **核心/后端**:存档时同时抓当前帧 → 缩略图(256×240 → 128×120 PNG);
  存档落盘到 `saves/<rom-hash>/slot-N.state` + `slot-N.png` + 元数据(时间戳/帧号)。
- **多槽位**:命名槽 + 自动存档(autosave 环)+ 快进倒带(rewind:环形缓冲每帧快照,可选)。
- **Tauri UI**:`n-grid` 卡片画廊(缩略图 + 时间 + 槽名),点击读取/覆盖/删除。

## 5. ROM 库(标准元数据 + 封面 + 预览)

- **库格式**:`library.json` —— 每条 `{ path, sha1, title, mapper, region, cover, added, last_played, play_secs }`。
- **扫描**:递归扫描 ROM 目录,解析 header,算 SHA1;可对接 No-Intro/数据库自动补标题/区域。
- **封面/预览**:封面图(用户设置 or 自动 = 无头跑 ~120 帧的标题画面截图);
  悬停预览 = 短 GIF/多帧轮播(无头渲染若干关键帧)。
- **Tauri UI**:封面网格 + 搜索/筛选(mapper/区域/收藏)、最近游玩、拖拽入库、右键编辑元数据。

## 6. 更多 Mapper(覆盖率冲 95%+)

优先级(按游戏覆盖):
1. **MMC2/MMC4**(9/10,标点小子)、**Color Dreams**(11)、**GxROM**(66)。
2. **MMC5**(5,银河战士改版/恶魔城3 日版,ExRAM/分屏/乘法器)。
3. **VRC** 系列(21/22/23/24/25/26,Konami,含 VRC IRQ 与音源)、**FME-7/Sunsoft-5B**(69)。
4. **Namco 163**(19)、**Mapper 71/232/79/113** 等长尾。
- 架构已就绪:实现 `MapperOps` + 工厂注册即可,不动 bus/ppu。

## 7. 硬件精度(向 FCEUX/Mesen 看齐)

- **PPU 逐点精确**:精灵取数分布到 dots 257–320(过 mmc3_test 3/4-A12/scanline_timing)、
  open-bus 衰减、OAM 衰减、调色板镜像边角、精灵溢出硬件 bug 精确化。
- **CPU/总线**:DMC DMA 周期窃取与 $2007/$4016 冲突(dmc_dma_during_read4)、
  OAM DMA 对齐、IRQ/NMI 边沿采样时刻。
- **区域**:PAL 16:5 精确分频、Dendy;制式由 header 自动选择。
- **音频**:更精确的非线性混音 LUT、低通/高通链路。
- **验收**:把 `nes-test-roms` 全套纳入 CI 回归,逐项打勾。

## 8. AI 能力 / MCP(差异化核心 —— 原生打通大模型与实时模拟器)

> 成熟模拟器只有"本地手动 + TAS 脚本";本项目把**内存读取、单帧步进、截图、按键注入、
> 反汇编、断点**封装为标准 MCP 工具,从核心层原生支持 LLM 实时交互,低延迟、一体化。

### 8.1 工具层扩展(`fc-mcp`)
- 现有 10 工具 → 增补:`emu_load_rom`、`emu_run_until`(到某 PC/帧/内存条件)、
  `emu_get_tilemap`(结构化 nametable)、`emu_set_speed`、`emu_list_breakpoints/add_breakpoint`、
  `emu_trace`(取执行追踪)、`emu_search_memory`(找血量/金钱变化,做"金手指自动发现")。
- 截图已是真 PNG;增加**结构化状态读取器**(可按游戏配置内存图谱 → 输出 score/lives/level)。

### 8.2 协议/集成
- 评估官方 `rmcp` crate 替换手写 JSON-RPC;支持 **stdio + HTTP/SSE** 两种传输。
- **三合一同实例**:让 Tauri 应用内嵌 MCP 端点,AI 与用户操作**同一个运行中的模拟器**
  (Agent 看到的就是用户屏幕),而非另起进程 —— 真正一体化、低延迟。

### 8.3 高层 Agent 闭环(可选独立 crate `fc-agent`)
- "暂停-思考-注入"模式:定时截图 → 调 Claude(多模态识图)→ 解析返回的按键序列 → 注入 → 循环。
- 应用:**自动闯关/解谜、自动试错、生成攻略**;**AI 辅助逆向**(读 CPU 汇编 + 断点追踪 +
  自动分析游戏逻辑/数据结构,产出标注与文档)。
- 用 `anthropic-sdk-rust` 做一个可选 demo,展示"AI 自己玩 1-1 / 自动找出血量地址"。

## 10. 架构原则(贯穿始终)

- `fc-core` 永远是纯逻辑、无 IO/无渲染;所有能力(金手指/断点/调试视图)在 core 实现,
  CLI/egui/Tauri/MCP 共享同一套 API —— **一次实现,四端复用**。
- 新增能力优先以"`ControlDeck` 公共方法 + 数据结构"暴露,前端只做展示与编排。
- 准确度以 `nes-test-roms` 客观验收,**零针对单游戏的 hack**。
